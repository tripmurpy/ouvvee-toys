# Agent Harness

## Urutan Kerja Agen
1. Baca `AGENTS.md`.
2. Baca `brain/about/project-context.md`.
3. Baca laporan terbaru di `brain/report/`.
4. Cek file yang relevan sebelum mengubah kode.
5. Setelah perubahan signifikan, tambah laporan sesi baru tanpa menimpa laporan lama.

## Batasan
- Gunakan solusi yang sudah ada di repo sebelum menulis kode baru.
- Jangan tambah dependency tanpa kebutuhan nyata.
- Simpan keputusan penting di laporan sesi.

## Pemeriksaan Minimum
- Untuk perubahan non-trivial, tinggalkan satu check runnable paling kecil.
